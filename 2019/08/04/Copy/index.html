Assignment vs Shallow Copy vs Deep Copy

2019-08-04

Python Object

Every datatype in python is object. No matter numbers, string, method or even module.

Every object in Python has three properties: identity, type and value.

There are two types of object:

  • Mutable object: Dict, List,Set. These objects could change value without changing identity。
  • Immutable object： Numbers, String, Tuple. Both identity and value of the object couldn’t change. Variable would refer to new created object and the old object will be garbage-collected.

Assignment

Assignment statements in Python do not copy objects, they create bindings between a target and an object. When use =, it only creates a new variable that shares the reference of the original object other than create a new object.

Sometimes a user wants copies that user can modify without automatically modifying the original at the same time, in order to do that we create copies of objects.

Shallow copy

A shallow copy constructs a new compound object and then (to the extent possible) inserts references into it to the objects found in the original. In shallow copy, the change of copy object will reflect in the original object.

Deep copy

Deep copy first construct a new collection object and then recursively populate it with copies of the child objects found in the original. The change of copy object will not reflect in the original object.

is vs ==

  • is compares the memory address of two objects
  • == compares the value of two objects.
