Data Mining (1)-- Basics

2019-09-08

Data and Knowledge

  • Data： A collection of facts about certain group of objects.
  • Pattern: Certain characteristics of data that are frequently observed.
  • Knowledge: Some general rules about the objects.

Data mining and KDD

KDD stands for Knowledge Discovery in Database. It is a process of discovering useful knowledge from big collections of data.

KDD is a process of identifying Valid, Novel, potentially useful and understandable patterns in data and deriving knowledge from the,

Data Mining is a step of KDD process in which interesting patterns are discovered. Some of these patterns are then interpreted and transformed into useful knowledge.

A user needs to decide what kinds of patterns he is interested and to pick a particular data mining algorithm to extract patterns.

A traditional DB usually only provides the function of storing and retrieving facts.

Result of Data Mining

The knowledge resulting from data mining should carry certain degree of predictive ability or descriptive ability.

Prediction involves using the database records that describe information about past behavior to automatically generate a model that can predict future behavior.(Example, Classification)

Description involves deriving patterns that summarize the underlying relationships in data and to describe the characteristics of data. (Example, clustering)

Data Warehousing

A data warehouse is an integration of various departmental database so that accesses to organization-wide data is possible.

A data warehouse is a convenient place where KDD and data mining applications are performed.

OLAP

The OLAP approach allows users to view data in multi-dimensional model(a Data Cube), supporting fast aggregation and summarization operations.

A process of Selection-> Group By-> Summarization.

Data Mining Tasks:

  • Classification
  • Clustering
  • Association Analysis
  • Regression

Data Types

Collection of data objects and their attributes. Object is also known as record, entity, tuple. An attribute is a property or characteristic of an object. A collection of attributes describe an object.

Types of Attributes:

  • Binary: Values are either 0 or 1, used to denote the presence of absence of a feature. Example: words in a document.

  • Nominal: Values are (discrete) names. Example: ID numbers, eye color.

  • Ordinal: Values are discrete names that are ordered. Example: rankings, grades.

  • Numerical: Values are numbers with which arithmetic operators can be applied. Example: height, speed.

  • Asymmetric attribute: Attributes that only presence values are important. Example, 1 for item purchased.

Types of data

  • Record(structured): Database tables.

  • Document data( sparse vectors)

    Each document is modeled as a vector of terms. Each term is a component(attribute) of the vector(document).

    • TF(term-frequency) model: value of each component is the number of times the corresponding term occurs in the document. Term vectors could be used to measure the similarity of documents.
    • Problem with TF-model: frequently occurring words dominate the vector.
    • TF-IDF model:

  • Set-valued Data:(Transactional Data)

    Each record is a set of items, example: market-basket data.

  • Graph: WWW, Social network

  • Spatial-temporal: Spatial data(location),Time series data, sequence data.

  • Other unstructured data: Example: images, sound files, video.

    • Extracting features from them( e.g. color histogram’s, frequency, etc.)
    • Representing them as record(structured) data based on features they contain.
    • Other descriptive data, such as tags, text descriptions.
    Data Quality
Data Qualify

Data quality problems:

  • Noise(incorrect data) and outliers(unusual data)

    Noise in data occurs due to error in data collection(e.g. a student of age 123 years old)

    Outliers are data that deviate so much from the norm that they may represent some rare of exceptional cases. Outliers are not necessarily noise.

  • Detecting Outliers: One method to detect outliers is to perform cluster analysis. Points that do not fall into any cluster could be flagged as outliers

    • Handling Noise: Perform outlier detection and then check manually whether the outliers are likely noise.

  • missing value

    Possible actions handle missing values

    • Ignore records with missing value(if only a small fraction of fata contains missing values)

  • Go and find missing data

    • Guess, fill in the missing value with average, major.

  • Duplicate and inconsistent data
