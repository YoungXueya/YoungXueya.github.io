Spark Streaming

1. Input of Spark Streaming application.

Spark Streaming takes input data stream from Kafka, Apache Flume, Amazon Kinesis, Twitter or TCP sockets.

Spark Dstream represents a stream of data divided into small batches. Internally, a Dstream is a represented as a sequence of RDDs.

2. Spark Streaming mechanism

Spark Streaming receives live input data, divided input data into batches, which then processed by the Spark engine to generate the final stream of result in batches.

In Receiver mode

1) KafkaReceiver (a long task running on a executor ) divides input stream into blocks and keeps in memory. Using blockManager to store data in executors. Report blockID to ReceiverTacker in Driver.

2). ReceiverTracker transform received data into RDD according to BlockID. Block=>RDD.

3) Spark Streaming start a job for each RDD, job executed by spark engine.

4) Task Scheduler of Spark engine schedules tasks to each executor. RDD partition => task.

3. Spark Streaming Configuration

  • spark.streaming.batchInterval: Interval that received RDD will be divided into batches. Usually 2s.

  • spark.streaming.blockInterval : Interval at which data received by Spark Streaming receivers is chunked into blocks of data before storing them in Spark. Default 200ms, minimum recommended 50ms.

  • spark.streaming.unpersist: Force RDDs generated and persisted by Spark Streaming to be automatically unpersisted from Spark’s memory(If true). This would cause RDD not found problem.
